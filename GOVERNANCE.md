# Governance

## Project Stewardship

detect-backend-threat is an open-source project maintained under the Apache 2.0 license.

## Roles

| Role | Responsibilities |
|---|---|
| **Maintainer** | Review and merge PRs, cut releases, manage infrastructure, enforce code of conduct |
| **Committer** | Submit PRs, review peer PRs (no merge rights), triage issues |
| **Contributor** | Submit bug reports, feature requests, documentation, code via pull requests |

## Maintainers

See [MAINTAINERS.md](MAINTAINERS.md) for the current list.

## Decision Making

- **Minor changes** (bug fixes, docs, small features): any Maintainer may merge after one approval.
- **Major changes** (API breaks, new integrations, architecture changes): require two Maintainer approvals and a 72-hour comment window.
- **Governance changes**: require unanimous Maintainer consensus and a 7-day comment window on the proposal issue.

## Becoming a Committer

Any contributor with at least 5 merged PRs of meaningful scope may be nominated by a Maintainer. The existing Maintainers vote; a simple majority approves.

## Becoming a Maintainer

Any Committer who has been active for at least 6 months may be nominated. Requires unanimous approval from existing Maintainers.

## Removing a Maintainer

A Maintainer who has been inactive for 6+ months or who violates the Code of Conduct may be removed by a majority vote of the remaining Maintainers.

## Conflict Resolution

Disputes are resolved by Maintainer vote. In the event of a tie, the original author (vigneshwar) holds a casting vote.

## Meetings

Maintainers communicate asynchronously via GitHub issues and pull requests. Synchronous meetings are held on an as-needed basis and notes are posted to the repository wiki.

## Amendments

This document may be amended via a pull request following the governance-change process above.
